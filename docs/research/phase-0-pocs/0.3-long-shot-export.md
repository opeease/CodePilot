# POC 0.3 — Artifact 长图导出（去 4000px 上限 + HTML/JSX 双输入）

**结论先行：** 复用现有 hidden BrowserWindow 安全模型（session partition / sandbox / no preload / block navigation），**核心扩展 = 去掉 `setSize` 高度上限 + 新增"分段 capturePage 拼接（pngjs）"与"CDP `Page.captureScreenshot { captureBeyondViewport:true }`"双路径**。拼接首选 **pngjs**（纯 JS，零原生依赖，`bitblt` 天然适合垂直拼接）；CDP 路径作为首选快速通道，OOM/canvas 超限时自动回退分段。JSX 输入走 **esbuild-wasm 预编译 + 独立 HTML** 路径（X-jsx-1），不依赖跨域读 iframe。

## [路径对比]

### 全页导出：X (分段拼接) vs Y (CDP captureBeyondViewport)

| 维度 | X：分段 `capturePage` + pngjs 拼接 | Y：CDP `Page.captureScreenshot { captureBeyondViewport:true }` |
|---|---|---|
| 实现复杂度 | 中（循环 + bitblt） | 低（一次调用） |
| 内存峰值 | 低（每段 width×4000 释放后再下一段） | 高（一次性编码整页 PNG） |
| 稳定性边界 | Chromium skia 单段 <16384px 即安全；整图总高由 pngjs 输出限制（理论受 Buffer/pixel data 长度，20000px 实测安全） | 大页面（>~30000px 或总像素 >2^29）易 OOM 崩 renderer |
| DevTools 冲突 | 无 | `debugger.attach` 与 DevTools 互斥（见 Electron 文档） |
| 依赖 | `pngjs`（纯 JS，0 原生） | Electron 内置 debugger |
| 失败模式 | 分段错位（y 偏移算错）/ 内存碎片 | `debugger_busy` / `oom` / CDP 协议超时 |

**决策：** 两条都实现，默认 **Y 优先**（快、简单）；检测到 `isDevToolsOpened()` 或 `contentHeight > Y_MAX_HEIGHT (= 16000)` 时回退 **X**；Y 抛 OOM 时也回退 X。

### JSX 导出：X-jsx-1 (预编译独立 HTML) vs X-jsx-2 (iframe 注入 html-to-image)

| 维度 | X-jsx-1：esbuild-wasm 预编译 → 独立 HTML | X-jsx-2：Sandpack iframe 注入 html-to-image + postMessage |
|---|---|---|
| 跨域可行性 | **完全绕过**（不读 iframe，独立窗口渲染） | 受 Sandpack iframe `allow-same-origin` 被禁限制；注入脚本须通过 Sandpack 支持的入口（`files` 或 `customSetup.entry`）间接执行 |
| 实现步骤 | 1) 渲染进程收 `files: Record<string,string>` 2) `esbuild-wasm` `build({ entryPoints:[entry], bundle:true, jsx:'automatic', write:false })` 3) 拼 `<html><head><script src="tailwind cdn"/></head><body><div id="root"/><script>${bundled}</script></body></html>` 4) 走 HTML 导出通道 | 1) 往 Sandpack `files` 追加 `__export.ts`（`import { toPng } from 'html-to-image'; toPng(document.getElementById('root')).then(d=>parent.postMessage({type:'artifact:png',data:d},'*'))`）2) 主文件末尾 import 它 3) 渲染进程 `window.addEventListener('message', ...)` 收 base64 |
| 风险 | 别名解析（`@/...`）需自行写 resolver plugin；Tailwind 需 runtime CDN 版本或预生成 CSS；缺失依赖（如 `react-dom/client`）需 importmap 或 bundle 内注入 | Sandpack iframe CSP 可能拦截 `postMessage` 跨域 target；`html-to-image` 对 SVG/canvas/外链字体不完美；长内容 dataURL > 2MB 时 postMessage 序列化慢 |
| 保真度 | 高（原生 Chromium 渲染） | 中（html-to-image 是 DOM→SVG foreignObject，字体/CSS filter/滤镜易失真） |
| 与现有通道复用 | **可复用** HTML 导出路径（X/Y） | 独立链路，需单独维护 |

**决策：** **X-jsx-1 为主**（复用全部基础设施 + 保真）；X-jsx-2 作为降级选项（不需要 esbuild-wasm 加载，1.2MB wasm 体积可省，但保真度降级）。

## [拼接库决策]

**推荐顺序：pngjs（采用）> nativeImage 组合（不可行）> sharp（不推荐）**

1. **pngjs ✅** — 零原生依赖，不影响 `scripts/after-pack.js`；`bitblt` 是为分块贴图设计的；整体加在 deps，ASAR 打包无特殊处理。
2. **nativeImage 组合 ❌** — 文档未暴露 compositing API（见上）；`createFromBitmap` 只能单张构造，无法粘贴其他 NativeImage；走 `toBitmap()` → 手动 Buffer 拼接 → `createFromBitmap()` 也要自己写像素数组，等于重实现 pngjs 的功能但没有 PNG 编码。
3. **sharp ❌** — 原生依赖，需要：(a) `scripts/after-pack.js` 加 `sharp` 到重编列表；(b) `electron-builder.yml` 加 `asarUnpack`；(c) macOS arm64+x64 + Windows x64 三套 prebuilt 在 CI 拉取；(d) 体积 ~30MB。**收益只是略快的 PNG 编码**，不匹配本需求（拼接频率低、非性能瓶颈）。

## [IPC 设计]

### 签名

```typescript
// electron/main.ts（新增，与 widget:export-png 并存）
type ExportSource =
  | { kind: 'html'; html: string }
  | { kind: 'sandpack-files'; files: Record<string, string>; entry: string; /* precompiled:true means files.entry already contains bundled JS HTML */ precompiled?: boolean };

interface ExportLongShotRequest {
  source: ExportSource;
  width: number;              // CSS px，必填；height 自动算
  pixelRatio?: number;        // 默认 2（HiDPI）
  outPath?: string;           // 不传则返回 base64；传了则写盘返回路径
  maxHeightPx?: number;       // 硬上限保护，默认 30000
  timeoutMs?: number;         // 默认 60000
}

type ExportLongShotError =
  | 'timeout'             // scriptsReady 或 capture 超时
  | 'canvas_limit'        // contentHeight > maxHeightPx
  | 'oom'                 // CDP 或 capturePage 崩溃/拒绝
  | 'debugger_busy'       // debugger.attach 抛异常
  | 'compile_error'       // esbuild-wasm 失败（仅 sandpack-files）
  | 'busy';               // 队列锁已持有

type ExportLongShotResult =
  | { ok: true; path?: string; base64?: string; bytes: number; width: number; height: number; route: 'cdp' | 'tiled' }
  | { ok: false; error: ExportLongShotError; detail?: string };

// 通道名
ipcMain.handle('artifact:export-long-shot', async (_e, req: ExportLongShotRequest): Promise<ExportLongShotResult> => { /* ... */ });
```

### 队列锁

模块级 `let exportLock: Promise<void> | null = null`；入口 `if (exportLock) return { ok:false, error:'busy' }`；`exportLock = task(); try { await exportLock } finally { exportLock = null }`。**主进程一次只允许一个 export window**，防内存翻倍。

## [实现草案]

```typescript
// electron/main.ts — 追加（仅草案，勿提交）
import { PNG } from 'pngjs';

const LONG_SHOT_Y_MAX = 16000;       // 超过走分段
const LONG_SHOT_TILE_H = 4000;       // 单段高度，与旧路径一致
let exportLock: Promise<unknown> | null = null;

ipcMain.handle('artifact:export-long-shot', async (_e, req: ExportLongShotRequest): Promise<ExportLongShotResult> => {
  if (exportLock) return { ok: false, error: 'busy' };
  const task = runExport(req);
  exportLock = task;
  try { return await task; } finally { exportLock = null; }
});

async function runExport(req: ExportLongShotRequest): Promise<ExportLongShotResult> {
  const { source, width, pixelRatio = 2, maxHeightPx = 30000, timeoutMs = 60000 } = req;

  // 1) 归一化为 HTML
  let html: string;
  try {
    html = source.kind === 'html' ? source.html : await compileSandpackToHtml(source);
  } catch (e) {
    return { ok: false, error: 'compile_error', detail: String(e) };
  }

  // 2) 开 hidden window（复用现有安全配置）
  const win = new BrowserWindow({
    show: false,
    width,
    height: LONG_SHOT_TILE_H,
    webPreferences: {
      contextIsolation: true,
      nodeIntegration: false,
      sandbox: true,
      partition: `export-${Date.now()}`,
      zoomFactor: pixelRatio, // HiDPI
    },
  });
  win.webContents.on('will-navigate', (e) => e.preventDefault());
  win.webContents.setWindowOpenHandler(() => ({ action: 'deny' }));

  const cleanup = () => { try { win.destroy(); } catch {} };
  const deadline = Date.now() + timeoutMs;

  try {
    await win.loadURL(`data:text/html;charset=utf-8,${encodeURIComponent(html)}`);

    // scriptsReady 等待（复用 console-message 约定）
    await waitScriptsReady(win, Math.max(1000, deadline - Date.now()));
    await delay(300);

    const contentHeight: number = await win.webContents.executeJavaScript('document.body.scrollHeight');
    if (contentHeight > maxHeightPx) { cleanup(); return { ok: false, error: 'canvas_limit', detail: `${contentHeight}>${maxHeightPx}` }; }

    // 3) 路由：CDP 优先，超限/DevTools 占用时回退分段
    const mainWin = BrowserWindow.getAllWindows().find(w => w !== win && !w.isDestroyed());
    const devtoolsOpen = mainWin?.webContents.isDevToolsOpened() ?? false;
    const route: 'cdp' | 'tiled' = (!devtoolsOpen && contentHeight <= LONG_SHOT_Y_MAX) ? 'cdp' : 'tiled';

    let png: Buffer;
    try {
      png = route === 'cdp'
        ? await captureViaCdp(win, width, contentHeight)
        : await captureTiled(win, width, contentHeight);
    } catch (err: any) {
      if (route === 'cdp' && /detached|target closed|out of memory/i.test(String(err?.message))) {
        // 回退一次
        png = await captureTiled(win, width, contentHeight);
      } else if (/debugger/i.test(String(err?.message))) {
        cleanup(); return { ok: false, error: 'debugger_busy', detail: String(err) };
      } else {
        cleanup(); return { ok: false, error: 'oom', detail: String(err) };
      }
    }

    // 4) 输出
    if (req.outPath) {
      await fs.promises.writeFile(req.outPath, png);
      cleanup();
      return { ok: true, path: req.outPath, bytes: png.length, width, height: contentHeight, route };
    }
    cleanup();
    return { ok: true, base64: png.toString('base64'), bytes: png.length, width, height: contentHeight, route };
  } catch (err: any) {
    cleanup();
    if (Date.now() >= deadline) return { ok: false, error: 'timeout' };
    return { ok: false, error: 'oom', detail: String(err?.message ?? err) };
  }
}

async function captureViaCdp(win: BrowserWindow, width: number, height: number): Promise<Buffer> {
  win.webContents.debugger.attach('1.3');
  try {
    const res = await win.webContents.debugger.sendCommand('Page.captureScreenshot', {
      format: 'png',
      captureBeyondViewport: true,
      clip: { x: 0, y: 0, width, height, scale: 1 },
      optimizeForSpeed: false,
      fromSurface: true,
    }) as { data: string };
    return Buffer.from(res.data, 'base64');
  } finally {
    try { win.webContents.debugger.detach(); } catch {}
  }
}

async function captureTiled(win: BrowserWindow, width: number, height: number): Promise<Buffer> {
  const tileH = LONG_SHOT_TILE_H;
  win.setSize(width, tileH);
  await delay(80);

  // 每段需滚动 + capturePage(rect)；rect 以 viewport 坐标为准 → 滚到对应 scrollTop 后 rect 取 (0,0,width,tileH)
  const segs: { y: number; buf: Buffer }[] = [];
  for (let y = 0; y < height; y += tileH) {
    const h = Math.min(tileH, height - y);
    await win.webContents.executeJavaScript(`window.scrollTo(0, ${y}); void 0;`);
    await delay(120);
    const img = await win.webContents.capturePage({ x: 0, y: 0, width, height: h });
    segs.push({ y, buf: img.toPNG() });
  }

  // pngjs 垂直拼接
  const canvas = new PNG({ width, height });
  for (const seg of segs) {
    const part = PNG.sync.read(seg.buf);
    part.bitblt(canvas, 0, 0, part.width, part.height, 0, seg.y);
  }
  return PNG.sync.write(canvas);
}

async function waitScriptsReady(win: BrowserWindow, timeoutMs: number): Promise<void> {
  return new Promise<void>((resolve) => {
    let done = false;
    const finish = () => { if (!done) { done = true; resolve(); } };
    win.webContents.on('console-message', (_e, _l, msg) => { if (msg === '__scriptsReady__') finish(); });
    setTimeout(finish, Math.min(timeoutMs, 6000));
  });
}

async function compileSandpackToHtml(src: Extract<ExportSource, { kind: 'sandpack-files' }>): Promise<string> {
  if (src.precompiled) return src.files[src.entry];
  // 在主进程跑 esbuild（非 wasm 即可，已在 devDeps），生产需加 esbuild 到 deps
  const esbuild = await import('esbuild');
  const result = await esbuild.build({
    stdin: { contents: src.files[src.entry], loader: 'tsx', resolveDir: '/' },
    bundle: true, format: 'iife', jsx: 'automatic', write: false,
    plugins: [{
      name: 'virtual-fs',
      setup(b) {
        b.onResolve({ filter: /.*/ }, (args) => {
          const p = args.path.replace(/^\.\//, '');
          if (src.files[p]) return { path: p, namespace: 'vfs' };
          return null; // 交给 esbuild 原生解析（node_modules 不存在 → 报错）
        });
        b.onLoad({ filter: /.*/, namespace: 'vfs' }, (args) => ({
          contents: src.files[args.path],
          loader: args.path.endsWith('.tsx') || args.path.endsWith('.jsx') ? 'tsx' : 'ts',
        }));
      },
    }],
  });
  const js = result.outputFiles[0].text;
  return `<!doctype html><html><head><meta charset="utf-8"><script src="https://cdn.tailwindcss.com"></script></head><body><div id="root"></div><script>${js}<\/script><script>setTimeout(()=>console.log('__scriptsReady__'),200)<\/script></body></html>`;
}

const delay = (ms: number) => new Promise(r => setTimeout(r, ms));
```

**注意：** 草案里 `compileSandpackToHtml` 在主进程跑 esbuild（native，已有 devDep），比 esbuild-wasm 简洁；若需保持渲染进程预编译（防主进程阻塞），改成渲染进程用 `esbuild-wasm` 编译完把 html 直接传过来即可（调用方传 `{kind:'sandpack-files', precompiled:true, files:{ 'index.html': html }, entry:'index.html' }`，主进程直通）。

## [DevTools 冲突]

- `debugger.attach` 与 DevTools 互斥仅对同一 `webContents` 生效。export window 独立，**不会** 影响主窗口 DevTools；但若主窗口 DevTools 已开，在导出 window 上 attach 也是安全的（两个 webContents 不共享）。
- 草案中仍做 `mainWin.isDevToolsOpened()` 兜底检查，是为"极端防御"：某些插件（如 React DevTools）可能通过全局 hook 干扰 CDP 序列化，出问题先走分段。
- 结论：**独立 hidden export window 策略足够隔离**，不影响用户日常 DevTools。

## [验收]

Phase 3 落地 smoke test（手动）：

1. **20000px HTML 长图** — 构造 `<div style="height:20000px;background:linear-gradient(#09f,#f09)">` 灌入 `artifact:export-long-shot`，产物 PNG `height=20000` 且上下端颜色正确、中段无错位条纹。
2. **HTML @ 30000px 触 canvas_limit** — 设 `maxHeightPx:25000` + 30000px 内容 → 返回 `{ ok:false, error:'canvas_limit' }`。
3. **Sandpack JSX（X-jsx-1）** — 传 `{ kind:'sandpack-files', files:{ 'index.tsx': 'export default()=><div className="p-8 text-2xl">Hello</div>' }, entry:'index.tsx' }`，产物含 Tailwind `p-8` 渲染效果。
4. **DevTools 开着仍可导出** — 主窗口 `Cmd+Opt+I` 打开 → 触发导出 → 期望 `route:'tiled'`（分段路径）。
5. **队列锁** — 连续触两次 → 第二次立即返回 `{ ok:false, error:'busy' }`。
6. **超时** — `timeoutMs:500` + 卡死脚本 → 返回 `{ ok:false, error:'timeout' }`。
7. **双平台** — macOS arm64（primary）+ Windows 10（VM）手动跑 1/3/4；Linux 不作要求。
8. **内存回归** — Activity Monitor / Task Manager 观察 export window destroy 后 renderer 进程被回收（30s 内无残留 Helper）。

**硬约束验证：** 本 POC 未修改 `electron/main.ts`，未新增依赖到 `package.json`；Phase 3 真正落地时需 `npm install pngjs @types/pngjs`（若用 esbuild-wasm 路径另加 `esbuild-wasm`）。
