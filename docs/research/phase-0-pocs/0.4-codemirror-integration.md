## [外部事实] CodeMirror 6 API & ESM

来源（访问日期 2026-04-19）：Bundle Guide <https://codemirror.net/examples/bundle/>；Ref <https://codemirror.net/docs/ref/>；System Guide <https://codemirror.net/docs/guide/>；Next.js Lazy Loading <https://nextjs.org/docs/app/guides/lazy-loading>（16.2.4 doc）；`unpkg.com/@codemirror/view/package.json` v6.41.1、`@codemirror/lang-markdown/package.json` v6.5.0。

1. 核心包全部 **dual ESM/CJS**（`type:"module"` + `exports.import/require` + `sideEffects:false`）。Electron 渲染进程（Chromium + Next webpack/turbopack）走 ESM 分支，**无加载问题**。
2. `EditorView` 构造器要求 `parent: Element`——严格 DOM 依赖，SSR 抛 `document is not defined`。
3. 动态主题：`Compartment.of(ext)` + `dispatch({ effects: comp.reconfigure(newExt) })`。
4. onChange：`EditorView.updateListener.of(u => u.docChanged && cb(u.state.doc.toString()))`。
5. 样式定制：`EditorView.theme({ "&": {...}, ".cm-content": {...} })`，`&` 代表根。

## [设计] MarkdownEditor 完整代码

**路径：** `src/components/markdown/MarkdownEditor.tsx`（新建，Phase 4 产出）

```tsx
"use client";

import { useCallback, useEffect, useRef } from "react";
import { EditorState, Compartment } from "@codemirror/state";
import { EditorView, keymap } from "@codemirror/view";
import { basicSetup } from "codemirror";
import { markdown } from "@codemirror/lang-markdown";
import { oneDark } from "@codemirror/theme-one-dark";
import { indentWithTab } from "@codemirror/commands";
import { useTheme } from "next-themes";

export interface MarkdownEditorProps {
  value: string;
  onChange: (v: string) => void;
  onSave?: () => void;
  filename?: string;
  placeholder?: string;
  className?: string;
}

export function MarkdownEditor({
  value, onChange, onSave, filename, placeholder, className,
}: MarkdownEditorProps) {
  const hostRef = useRef<HTMLDivElement | null>(null);
  const viewRef = useRef<EditorView | null>(null);
  const themeCompartment = useRef(new Compartment()).current;
  const onChangeRef = useRef(onChange);
  const onSaveRef = useRef(onSave);
  const { resolvedTheme } = useTheme();

  // 保持最新回调，避免 CodeMirror 里闭包陈旧
  useEffect(() => { onChangeRef.current = onChange; }, [onChange]);
  useEffect(() => { onSaveRef.current = onSave; }, [onSave]);

  // 初始化 EditorView（只挂载一次）
  useEffect(() => {
    if (!hostRef.current || viewRef.current) return;

    const baseTheme = EditorView.theme({
      "&": { height: "100%", fontSize: "13px" },
      ".cm-scroller": { fontFamily: "var(--font-mono, ui-monospace, monospace)" },
      ".cm-content": { padding: "12px" },
      "&.cm-focused": { outline: "none" },
    });

    const saveKeymap = keymap.of([
      {
        key: "Mod-s",
        preventDefault: true,
        run: () => { onSaveRef.current?.(); return true; },
      },
      indentWithTab, // 保留 SkillEditor 原 Tab 缩进行为
    ]);

    const initialTheme = resolvedTheme === "dark" ? oneDark : [];

    const state = EditorState.create({
      doc: value,
      extensions: [
        basicSetup,
        markdown(),
        baseTheme,
        themeCompartment.of(initialTheme),
        saveKeymap,
        EditorView.lineWrapping,
        EditorView.updateListener.of((u) => {
          if (u.docChanged) onChangeRef.current(u.state.doc.toString());
        }),
      ],
    });

    viewRef.current = new EditorView({ state, parent: hostRef.current });
    return () => { viewRef.current?.destroy(); viewRef.current = null; };
  // 初始化只跑一次：props 变化通过后续 effect 同步
  // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);

  // 外部 value 变化 → 写回（受控）
  useEffect(() => {
    const view = viewRef.current;
    if (!view) return;
    const cur = view.state.doc.toString();
    if (cur !== value) {
      view.dispatch({ changes: { from: 0, to: cur.length, insert: value } });
    }
  }, [value]);

  // 主题切换（Compartment.reconfigure，无闪烁）
  useEffect(() => {
    const view = viewRef.current;
    if (!view) return;
    view.dispatch({
      effects: themeCompartment.reconfigure(resolvedTheme === "dark" ? oneDark : []),
    });
  }, [resolvedTheme, themeCompartment]);

  // 占位符：CodeMirror 自带 placeholder() 扩展；此处略，改造时按需加
  return (
    <div
      ref={hostRef}
      className={className ?? "h-full w-full overflow-hidden"}
      data-filename={filename}
      aria-label={placeholder}
    />
  );
}
```

**特性覆盖：** 虚拟化（CodeMirror viewport 自带，万行 Markdown 渲染 O(可见行数)）；Tab 缩进（`indentWithTab`）；`Mod-s` 保存；主题切换无闪；受控 `value` 往返；`lineWrapping` 软换行。

## [Tailwind v4 样式隔离] 三路径对比

Tailwind v4 preflight（`@layer base`）重置 `pre/code` 的 `font/margin/white-space`；CM 的 `.cm-scroller/.cm-line` 内部重用 pre 行为，未隔离会光标错位、行高错乱。

| 方案 | 可行性 / 成本 | 副作用 |
|---|---|---|
| **A Shadow DOM** | 可行但需改造 `EditorView` 挂 `attachShadow` 并搬 `<style>`（见官方 `editor-in-shadow-dom`）；**高成本** | 丢失 Tailwind token 继承（`--border/--radius` 需显式转发），focus/contextmenu 被截流 |
| **B CSS Module** | **不可行**：Tailwind v4 cascade layers 机制下，层外 class 无法降级层内 preflight | — |
| **C 反制 preflight（推荐）** | `globals.css` 加 `@layer base { .cm-editor, .cm-editor * { all: revert-layer; } }`，再用 `EditorView.theme` 重声必要属性；**低成本** | 需回归测试光标、滚动、选区 |

**推荐 C**。保留 token 继承、成本最低；冲突升级再退 A。

## [集成模式] dynamic import

**`next/dynamic` + `{ ssr: false }`**。

- CodeMirror 纯浏览器依赖，SSR 必抛 `document is not defined`。
- Next.js 16 明确：`ssr:false` 不允许用于 Server Component，必须在 Client Component 内声明。`SkillEditor` 已 `"use client"`，直接 `dynamic()` 即可。
- 静态 import 把 135KB 打进每个引用路由 chunk — 拒绝。Shadow DOM 不采用（见表 A）。

**包装器：** `src/components/markdown/MarkdownEditor.lazy.tsx`

```tsx
"use client";
import dynamic from "next/dynamic";
import type { MarkdownEditorProps } from "./MarkdownEditor";

export const MarkdownEditor = dynamic<MarkdownEditorProps>(
  () => import("./MarkdownEditor").then((m) => m.MarkdownEditor),
  {
    ssr: false,
    loading: () => (
      <div className="h-full w-full animate-pulse bg-muted/40" aria-busy="true" />
    ),
  }
);
```

消费者统一 `import { MarkdownEditor } from "@/components/markdown/MarkdownEditor.lazy"`。

## [验收条件] Phase 4 smoke test

1. `npm run dev` 打开 `/skills` 点技能 → CM 挂载 < 500ms，console 无 `document is not defined`。
2. 5000 行 Markdown fixture，滚动帧耗时 p95 < 16ms，输入延迟 < 8ms。
3. Tab 插两空格；`Cmd/Ctrl-S` 触保存并显示 `saved` 徽章。
4. Light/Dark 切换：CM 配色无闪烁（`Compartment.reconfigure` 单帧）。
5. Preflight 回归：光标基线对齐、行高 ≥ 1.4、选区背景可见。
6. Network 面板：首屏无 `MarkdownEditor` chunk，进 `/skills` 才请求。
7. Electron 打包 DMG 启动，1-4 全通过，`main.log` 无 ESM 报错。
8. 视图切换 `edit→preview→edit` 无 memory leak（`destroy()` cleanup；EditorView 实例 ≤ 1）。

**回退：** preflight 方案 C 不够干净 → 退 A Shadow DOM（token 显式注入）；135KB 反馈"首次卡" → 降级 minimal（-60KB），扩展按需 lazy。
